
# Jakarta Servlet Package

## Overview

The Jakarta Servlet package, part of the Jakarta EE (formerly Java EE) ecosystem, is used to build dynamic web applications in Java. 
Servlets are Java classes that handle HTTP requests and responses, acting as the backbone of Java-based web servers and applications.
This package provides a set of interfaces and classes that allow developers to create and manage servlets.

The main package for using servlets is `jakarta.servlet`, with the core classes and interfaces needed for building web applications.

## Key Classes and Interfaces

### `HttpServlet`
The `HttpServlet` class extends `GenericServlet` and provides methods to handle HTTP-specific requests, such as `doGet`, `doPost`, `doPut`, and `doDelete`. Each method corresponds to a different HTTP method, making it easier to create RESTful services.

### `ServletRequest` and `ServletResponse`
`ServletRequest` and `ServletResponse` provide access to HTTP request data and responses. With these classes, developers can read request data, such as headers and parameters, and write response data, like HTML content or JSON responses.

### `ServletContext`
`ServletContext` allows servlets to access web application parameters and manage application-level information, such as initialization parameters and shared resources.

## Example: Simple Servlet

Hereâ€™s an example of a basic servlet that processes HTTP GET requests:

```java
import jakarta.servlet.ServletException;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;
import java.io.PrintWriter;

public class HelloWorldServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        response.setContentType("text/html");
        PrintWriter out = response.getWriter();
        out.println("<h1>Hello, World!</h1>");
    }
}
```

In this example, the `HelloWorldServlet` class extends `HttpServlet` and overrides the `doGet` method to handle HTTP GET requests. When accessed, it sends a simple HTML response with "Hello, World!".

## Configuring and Deploying a Servlet

To deploy a servlet in a Jakarta EE web application:

1. **Add the Servlet to `web.xml`**  
   The `web.xml` file, located in the `WEB-INF` directory, configures servlets and mappings. Here's an example configuration:

   ```xml
   <web-app xmlns="https://jakarta.ee/xml/ns/jakartaee"
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
            xsi:schemaLocation="https://jakarta.ee/xml/ns/jakartaee https://jakarta.ee/xml/ns/jakartaee/web-app_4_0.xsd"
            version="4.0">

       <servlet>
           <servlet-name>HelloWorldServlet</servlet-name>
           <servlet-class>com.example.HelloWorldServlet</servlet-class>
       </servlet>

       <servlet-mapping>
           <servlet-name>HelloWorldServlet</servlet-name>
           <url-pattern>/hello</url-pattern>
       </servlet-mapping>
   </web-app>
   ```

   This configuration maps `HelloWorldServlet` to the `/hello` URL.

2. **Package and Deploy**  
   To deploy the servlet, package the web application as a WAR file and deploy it to a Jakarta-compatible server, such as Apache Tomcat or GlassFish.

   ```bash
   # Assuming you're using Maven
   mvn clean package
   ```

   The WAR file can then be copied to the `webapps` directory of Tomcat or another application server.

---

## Running the Servlet

1. Start your Jakarta EE server (e.g., Tomcat).
2. Deploy the WAR file containing the servlet.
3. Access the servlet by navigating to `http://localhost:8080/yourAppName/hello`.

If everything is configured correctly, you should see the "Hello, World!" message displayed in your browser.

## Advanced Topics

1. **Filters and Listeners**  
   Filters intercept requests and responses, allowing developers to modify them before they reach a servlet or after they leave it. Listeners monitor events in a web application, such as startup and shutdown.

2. **Session Management**  
   The Jakarta Servlet package provides `HttpSession` for tracking user sessions, allowing data to be stored across multiple requests.

3. **Error Handling**  
   Custom error pages can be specified in `web.xml`, and exceptions can be managed with custom handlers.
