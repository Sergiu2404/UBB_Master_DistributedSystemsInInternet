
# Jakarta Session Management in Servlets

## Overview

Session management is a core component of web applications in Jakarta EE. It allows applications to maintain stateful interactions with clients across multiple HTTP requests. Jakarta Servlet provides several ways to manage sessions effectively, typically through the `HttpSession` interface.

## Creating and Managing Sessions

In Jakarta Servlets, sessions are managed using the `HttpSession` object. When a client connects to the server, a session can be created or retrieved, allowing data to persist between requests.

### Example: Creating a Session

Here’s an example servlet that creates a session if one doesn’t already exist:

```java
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;
import java.io.IOException;

@WebServlet("/sessionExample")
public class SessionExampleServlet extends HttpServlet {

    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        HttpSession session = request.getSession(true); // Creates a session if one doesn't exist
        session.setAttribute("username", "testUser"); // Sets an attribute in the session
        response.getWriter().println("Session created with username: " + session.getAttribute("username"));
    }
}
```

In this example:
- `request.getSession(true)` creates a session if it doesn’t exist.
- `setAttribute` and `getAttribute` methods allow storing and retrieving data within the session.

## Retrieving Session Attributes

Sessions allow you to store attributes, which can be accessed across multiple requests. Attributes can be any Java `Object`.

### Example: Storing and Retrieving Attributes

In this example, we’ll store and retrieve a user ID within the session.

```java
// Storing an attribute
session.setAttribute("userId", 12345);

// Retrieving an attribute
Integer userId = (Integer) session.getAttribute("userId");
```

## Session Timeout and Invalidation

Sessions have a default timeout setting, typically 30 minutes, after which they are invalidated. You can configure session timeouts in `web.xml` or manually invalidate them.

### Example: Session Timeout

Set session timeout in `web.xml`:

```xml
<session-config>
    <session-timeout>15</session-timeout> <!-- 15 minutes -->
</session-config>
```

To invalidate a session manually:

```java
session.invalidate();
```
