# Java JPA Annotations

This document provides a list of the most commonly used Java JPA (Java Persistence API) annotations, along with a brief
description for each. These annotations are used to map Java classes and their fields to database tables and columns, e
nabling ORM (Object-Relational Mapping).

## Annotations List

### 1. `@Entity`

- **Description**: Specifies that the class is an entity and is mapped to a database table.
- **Usage**:

```java

@Entity
public class Product { ...
}
```

### 2. `@Table`

- **Description**: Specifies the table in the database that the entity maps to. Can be used to specify the schema as
  well.
- **Usage**:

```java

@Entity
@Table(name = "products", schema = "store")
public class Product { ...
}
```

### 3. `@Id`

- **Description**: Marks a field as the primary key for the entity.
- **Usage**:

```java
@Id
private Long id;
```

### 4. `@GeneratedValue`

- **Description**: Specifies that the value of the primary key will be generated automatically, often using strategies
  like AUTO, IDENTITY, SEQUENCE, or TABLE.
- **Usage**:

```java

@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;
```

### 5. `@Column`

- **Description**: Specifies the details of the column that maps to a field, including name, length, and whether it's
  nullable.
- **Usage**:

```java

@Column(name = "product_name", nullable = false, length = 50)
private String name;
```

### 6. `@OneToOne`

- **Description**: Defines a one-to-one relationship between two entities.
- **Usage**:

```java

@OneToOne
private ProductDetails productDetails;
```

### 7. `@OneToMany`

- **Description**: Defines a one-to-many relationship where one entity is related to a list of another entity.
- **Usage**:

```java

@OneToMany(mappedBy = "category")
private List<Product> products;
```

### 8. `@ManyToOne`

- **Description**: Defines a many-to-one relationship where multiple entities relate to a single entity.
- **Usage**:

```java
@ManyToOne
private Category category;
```

### 9. `@ManyToMany`

- **Description**: Defines a many-to-many relationship between two entities.
- **Usage**:

```java

@ManyToMany
private List<Tag> tags;
```

### 10. `@JoinColumn`

- **Description**: Specifies the foreign key column used in the relationship.
- **Usage**:

```java
@ManyToOne
@JoinColumn(name = "category_id")
private Category category;
```

### 11. `@JoinTable`

- **Description**: Specifies the join table for a many-to-many relationship.
- **Usage**:

```java

@ManyToMany
@JoinTable(
        name = "product_tag",
        joinColumns = @JoinColumn(name = "product_id"),
        inverseJoinColumns = @JoinColumn(name = "tag_id")
)
private List<Tag> tags;
```

### 12. `@Transient`

- **Description**: Marks a field that should not be persisted in the database.
- **Usage**:

```java

@Transient
private String tempData;
```

### 13. `@Lob`

- **Description**: Marks a field as a Large Object (LOB) type, used for large data types like BLOB or CLOB.
- **Usage**:

```java
@Lob
private byte[]image;
```

### 14. `@Enumerated`

- **Description**: Specifies how an enum is stored in the database (either as an ordinal or a string).
- **Usage**:

```java
@Enumerated(EnumType.STRING)
private ProductType productType;
```

### 15. `@Temporal`

- **Description**: Specifies the type of a date (DATE, TIME, or TIMESTAMP) for a java.util.Date field.
- **Usage**:

```java
@Temporal(TemporalType.DATE)
private Date manufactureDate;
```

### 16. `@Embedded`

- **Description**: Marks a field as an embeddable class.
- **Usage**:

```java
@Embedded
private Address address;
```

### 17. `@Embeddable`

- **Description**: Specifies that a class can be embedded in an entity.
- **Usage**:

```java

@Embeddable
public class Address { ...
}
```

### 18. `@AttributeOverride`

- **Description**: Allows overriding of attributes in an embeddable class.
- **Usage**:

```java
@Embedded
@AttributeOverride(name = "street", column = @Column(name = "home_street"))
private Address address;
```

### 19. `@Access`

- **Description**: Specifies the access type (FIELD or PROPERTY) for an entity or a specific field.
- **Usage**:

```java
@Access(AccessType.FIELD)
private String name;
```

### 20. `@NamedQuery`

- **Description**: Defines a static, named query that can be used across the application.
- **Usage**:

```java
@NamedQuery(name = "Product.findAll", query = "SELECT p FROM Product p")
```

### 21. `@Basic`
- **Description**: Specifies that a field is a basic property that should be persisted. It is typically optional because most fields are assumed to be basic by default. However, `@Basic` can be used to specify additional settings, such as whether the attribute is `optional` (nullable) and its fetch type (`EAGER` or `LAZY`).
- **Usage**:
  ```java
  @Basic(optional = false, fetch = FetchType.LAZY)
  private String description;
